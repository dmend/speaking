Nonrepudiation

Authenticity

  • Confidentiality + Integrity + Nonrepudiation

  • If you receive a message that is encrypted and signed, then you can be sure that the contents have been kept private, the message has not been tampered with and the message was sent by someone who has the right to send it in the senders name.

Symmetric Encryption

  • Symmetric because it uses the same key for both operations

[Plaintext] + [KEY] -> ENCRYPTION -> [Ciphertext]

[Ciphertext] + [KEY] -> DECRYPTION -> [Plaintext]

11/32

Asymmetric Encryption

  • Split KEY in two
    • KEY A
    • KEY B

[Plaintext] + [KEY A] -> ENCRYPTION -> [Ciphertext]

[Ciphertext] + [KEY B] -> DECRYPTION -> [Plaintext]

Public-Key Encryption

  • Same as Asymmetric encryption, but you give away one of the key halves

    • Public Key
    • Private Key

  • GPG uses keyservers to share public keys

GPG Uses Public-Key Encryption

GPG Keys

  • "Key" - File in your computer

  • Passphrase - Make sure it's not easy to guess!

  • GPG Uses two key pairs by default

    • Master key used for signing
    • Subkey used for encription
  • GPG Keys are tied to a User ID
    • Legal Name (Comment) email@addres.com
  • Fingerprint
  • Key ID (last 8 digits of fingerprint)

Trust

  • In PGP Trust is when you sign someone else's private key to affirm that you have verified that person's identity.

  • When someone signs your key, they are saying that they've confirmed your identity.

  • When you sign someone else's key, you're telling the world that you're positive of the identity of the key owner.
