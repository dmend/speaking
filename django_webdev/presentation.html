URLConf

  • Uses Regular Expressions for URL matching

  • You can pass regex groups as parameters to your view

     1 from django.conf.urls import patterns, include, url
 2 from django.contrib import admin
 3 
 4 admin.autodiscover()
 5 
 6 urlpatterns = patterns('',
 7     # http://bookstore.com
 8     url(r'^$', 'inventory.views.home'),
 9     # http://bookstore.com/book/12345
10     url(r'^book/(\d+)/$', 'inventory.views.detail'),
11     # http://bookstore.com/admin/*
12     url(r'^admin/', include(admin.site.urls)),
13 )

Models

  • Each model is a Python class that subclasses django.db.models.Model

Security - SQL Injection

  • By using the Django ORM you get protection from SQL Injection for free!
  • Values sent to the ORM are escaped based on your database backend

  • You can execute raw SQL if you need to, but be careful!

    1 # DON'T DO THIS!
2 >>> Book.objects.raw(
3     'SELECT * FROM myapp_book WHERE title = %s' % title
4 )
5 
6 # Do this instead
7 >>> Book.objects.raw(
8     'SELECT * FROM myapp_book WHERE title = %s', [title]
9 )

View Functions

  • Views define which data you see, not how you see it

     1 from django.http import HttpResponse, Http404
 2 from django.shortcuts import render_to_response
 3 
 4 from inventory.models import Book
 5 
 6 def detail(request, isbn):
 7     try:
 8         b = Book.objects.get(pk=isbn)
 9     except Book.DoesNotExist:
10         raise Http404
11     return render_to_response('detail.html', {'book': b})

  • Easy to use
  • Not on by default

    1 <form action="." method="post">
